# Game Report
### B12611050 蕭鎮緯
### game name: 亞洲人生存大挑戰
## 1. Description of my game
《亞洲人生存大挑戰》是一款以「亞洲家庭文化與生命壓力」為主題的互動式文字冒險遊戲。
玩家從出生開始，一路經歷七個關卡：
- 出生（性別開局）
- 大學志願（科系選擇）
- 第一份工作（社會起跑線）
- 結婚對象（愛情與家族 KPI）
- 生小孩（生與不生都是題）
- 過年大拷問（長輩玄學問候術）
- 親戚稱謂魔王關（華人家族樹的深淵)

遊戲的主軸是「亞洲傳統期待與個人選擇的拉扯」。
每一關玩家的選擇都會被換算成 HP，代表社會期待對個體的壓力。
此外，每一關都會生成一句人生小筆記——帶點幽默、靠北、自嘲，但也溫柔的金句。
這些筆記陪著玩家逐關前進，也成為最後人生回顧的一部分。
若 HP 歸零，遊戲立即結束；
若成功撐過七關，玩家會看到一篇 完全依照玩家選擇生成的個人化人生回顧。
這是我對亞洲人生劇本的一次詮釋：荒謬、溫柔、現實，笑中又帶著一點淚。

## 2. Details on the LLM Tasks Applied
遊戲中包含多個 LLM 應用，每一關都依賴不同的語言模型能力。以下為本遊戲實作的主要 LLM 任務：
### 2.1 情境生成
工作關、結婚關與親戚稱謂關皆透過 LLM 自動產生三個選項，情境描述包含家庭期待、社會壓力、人物特質等。
過年拷問則由 LLM 生成長輩提問，並附上難度標籤（low/medium/high/extreme）。
### 2.2 玩家回答分類
在過年大拷問中，玩家的回答會被丟給另一個 LLM 解析語氣，分類為：
- balanced
- bragging
- too_humble
- defensive
- refuse

分類結果影響 HP 的增減。
### 2.3 關卡結果敘述
每一關的結果文字（result）與人生小筆記（note）皆由 LLM 依據：
- 關卡背景
- 玩家選擇
- HP 變化
- 標籤（tag）

產生 100～200 字的情境敘述，以及一則風格固定的短句金句。

### 2.4 親戚稱謂推論
最後一關要求 LLM：
1. 生成一題親戚稱謂魔王題
2. 提供正確答案
3. 玩家輸入後，程式再判斷是否符合正確稱謂

這部分是本遊戲最具挑戰性的 LLM 任務。

### 2.5 人生總回顧
最後 LLM 根據玩家真實的logs(每關選項、人生小筆記、HP 變化）生成400～700字的人生回顧文。
回顧內容必須：
- 只能提到玩家實際經歷的關卡
- 不能腦補未經歷的事件
- 要溫柔、有畫面感
- 要帶出每一則玩家累積的人生小筆記

這部分是我最滿意的部分，起初他會有點腦補未來資訊（如果玩家沒有玩到底），也是後來prompt下得夠明確才成功。

## 3. My Discovery
在整個遊戲的開發與測試過程中，我注意到以下幾點：
### 3.1 使用 gpt-3.5 會遇到邏輯性限制
因為學校提供的 API key 限制只能使用 gpt-3.5 系列，我發現模型在處理複雜邏輯時穩定性不佳，例如親戚稱謂邏輯容易錯，gpt-3.5 在處理 3～6 層家族關係時常會產生不屬於華人文化的稱謂、遺漏輩分、把「舅、姨、伯、姑」搞混、明明應該是男性稱謂卻生成女性稱謂（反之亦然）。因此我特別設計了：答案合理性檢查 function（避免亂給詞）、兩次重試機制、Fallback 題目（確保遊戲不會壞掉）來解決這個問題（在不改變模型的條件下）
### 3.2 gpt-3.5 的 JSON 產出不穩定
模型常常：
在 JSON 前後加句子少一個或逗號就壞掉等等不遵守格式範例為此，我加入：call_llm_json()和JSON substring 提取與第二次 parse以及必要時 fallback 行為來大幅提高遊戲可玩度。
### 3.3 文字風格需明確限制
gpt-3.5 在敘事時過度「雞湯」或「太正經」。為了避免這種風格偏移，我在 prompt 中加入：指定語氣（微靠北、微自嘲、但溫柔）字數範圍、不可出現技術內容、提供示例但禁止重複、讓遊戲敘事更一致。
